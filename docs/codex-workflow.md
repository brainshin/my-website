# Codex 협업 가이드

Codex와 작업할 때는 수정 범위, 검증 방법, push 여부를 분명히 남기는 것이 좋습니다.

## 요청 예시

| 상황 | 좋은 요청 |
| --- | --- |
| 위치만 알고 싶을 때 | "수정하지 말고 파일 위치와 방법만 알려줘." |
| 직접 수정할 때 | "이 파일을 수정하고 테스트까지 확인해줘." |
| 배포까지 할 때 | "commit하고 main 브랜치에 push해줘." |

## 체크리스트

- 변경 전 `git status` 확인
- 수정 파일 범위 확인
- preview 또는 배포 결과 확인
- commit 메시지 작성
